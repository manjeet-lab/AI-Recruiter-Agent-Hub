This document details the architecture, design choices, implementation details,

--------------------------------------------------------------------------------
1. EXECUTIVE SUMMARY
--------------------------------------------------------------------------------
The AI Recruiter Agent Hub is a production-grade, secure, multi-agent AI system
designed to automate and elevate the end-to-end candidate resume screening process.
Powered by the Google Agent Development Kit (ADK) and the Gemini 2.5 Flash large 
language model, the system models the cognitive workflow of a professional recruiter.
The hub parses candidate PDF resumes, evaluates skills, experience, and education, 
calculates domain-specific recruiter and Applicant Tracking System (ATS) scores, 
and generates constructive feedback and a personalized 30-day career roadmap.

To deliver state-of-the-art security, all file storage, retrieval, and PDF parsing 
operations are fully decoupled from both the UI and the AI agents, instead routed 
exclusively through a secure, custom-built Model Context Protocol (MCP) Filesystem 
Server. The user experience is hosted in a premium, dark-mode Streamlit dashboard 
featuring progressive execution rendering and an instant, keyless demo mode. The 
result is an enterprise-ready screening tool that reduces initial screening time 
from hours to seconds while offering candidates transparent, actionable growth pathways.

--------------------------------------------------------------------------------
2. PROBLEM STATEMENT
--------------------------------------------------------------------------------
Modern recruiting is characterized by a severe volume bottleneck. A typical job 
posting attracts hundreds of applicants, yet industry studies show that recruiters 
spend an average of only 6 to 7 seconds scanning a resume before making an initial 
hire/reject decision. Under such pressure, hiring decisions are highly prone to 
cognitive biases, oversight, and noise. Outstanding candidates are frequently 
filtered out due to minor ATS formatting quirks or an absence of exact keywords, 
even when they possess equivalent skills.

Furthermore, the traditional candidate experience is notoriously opaque. Rejected 
candidates are almost never given constructive feedback, creating a demoralizing 
"black hole" recruitment loop. Recruiters want consistent, objective evaluations 
tailored to specific professional domains, and candidates want transparency and 
actionable advice on how to improve. The AI Recruiter Agent Hub solves both sides 
of this equation.

3. SOLUTION OVERVIEW
--------------------------------------------------------------------------------
The AI Recruiter Agent Hub introduces an intelligent, automated pipeline that 
simulates a thorough, human-like recruiting panel. The application operates in 
two core modes:

1. Instant Demo Mode (Offline Local Evaluation):
   Designed for immediate stakeholder evaluation without requiring API keys. It 
   loads pre-analyzed, rich mock candidate profiles (such as Sarah Chen, a Senior 
   Software Engineer, and Alex Rivers, a Biotech Research Associate) to demonstrate 
   the Streamlit dashboard's responsive scoring widgets, feedback reports, and 
   interactive 30-day career roadmaps.

2. Real Resume Analysis Mode (Real-Time AI Pipeline):
   Enables recruiters to upload a PDF resume (up to 5 MB), input a Gemini API 
   key, and run the full four-stage multi-agent pipeline. The system processes 
   the resume progressively, rendering the outputs of each agent stage in real-time 
   as they are generated.

For every analyzed candidate, the system delivers:
- A structured profile extracting contact information, skills, and work history.
- Symmetrical, category-specific recruiter card scores (0–100) and hiring decisions.
- A descriptive recruiter narrative outlining highlights and potential red flags.
- Actionable candidate feedback and a personalized 30-day learning and project roadmap.

--------------------------------------------------------------------------------
4. WHY MULTI-AGENT ARCHITECTURE?
--------------------------------------------------------------------------------
A monolithic prompt containing all evaluation steps ("parse this, score this, write 
feedback, and build a career plan") is unsuitable for high-stakes enterprise systems. 
It suffers from:
- Context Pollution: The model struggles to separate neutral text parsing from 
  nuanced scoring or creative career advising, leading to hallucinations.
- Output Instability: Generating multiple distinct JSON objects in a single pass 
  frequently violates structured schemas.
- Lack of Granular Control: Monolithic prompts cannot run different stages at 
  different temperatures (e.g., parsing needs to be highly deterministic, while 
  career recommendations benefit from creative reasoning).

By adopting a sequential, multi-agent architecture using the Google ADK framework, 
each sub-agent is assigned a singular, well-defined responsibility with its own 
system instructions and custom-tuned temperature. The outputs are validated against 
independent Pydantic models at each transition point, guaranteeing data integrity. 
The pipeline runs downstream using a cumulative context, where subsequent agents 
build on the strict, validated schemas of their predecessors.

--------------------------------------------------------------------------------
5. GOOGLE ADK ARCHITECTURE
--------------------------------------------------------------------------------
The application utilizes a hierarchical, coordinator-specialist agent architecture 
established via the Google Agent Development Kit (ADK). The Streamlit frontend 
interacts exclusively with a single orchestration module, abstracting all internal 
agent logic and execution states.

```
                  ┌─────────────────────────────┐
                  │        Streamlit UI         │
                  └──────────────┬──────────────┘
                                 │ (yields progressive stages)
                  ┌──────────────▼──────────────┐
                  │    Orchestrator Agent       │
                  └──────────────┬──────────────┘
                                 │
         ┌───────────────────────┼───────────────────────┐
         │ (Stage 1)             │ (Stage 2)             │ (Stage 3 & 4)
┌────────▼────────┐     ┌────────▼────────┐     ┌────────▼────────┐
│ Resume Analysis │     │    Recruiter    │     │   Feedback &    │
│      Agent      │     │  Scoring Agent  │     │ Recommendations │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

The multi-agent hierarchy consists of:

1. Orchestrator Agent (orchestrator_agent.py):
   The root of the ADK tree. It manages the sequential pipeline, routes inputs 
   and intermediate outputs, and interfaces with the MCP filesystem client to 
   read resumes and persist consolidated candidate reports. It is implemented 
   as a Python generator that yields results step-by-step, enabling the UI to 
   update progressively instead of making the user wait for the entire process.

2. Resume Analysis Agent (resume_analysis_agent.py):
   Runs at a highly deterministic temperature of 0.1. It accepts plain text from 
   the resume and extracts a structured JSON profile mapping to `ResumeSchema` 
   (Pydantic). Crucially, it infers the candidate's primary "Professional Domain" 
   (e.g., Software Engineering, Data Science, Biotechnology, Finance, Marketing, 
   or General Professional), which dictates all downstream scoring rules.

3. Recruiter Scoring Agent (scoring_agent.py):
   Runs at a temperature of 0.2. It performs a domain-aware evaluation, matching 
   the candidate's profile against industry-specific guidelines. For example, 
   a software engineer is evaluated on coding, system design, and GitHub projects, 
   whereas a biotech candidate is scored on laboratory techniques, publications, 
   and GMP compliance. It scores five categories (Technical Skills, Projects, 
   Experience, Communication, Education) and issues a final hiring decision 
   ("Hire", "Maybe Hire", "Reject") along with highlights, reasoning, and red flags.

4. Feedback Agent (feedback_agent.py):
   Runs at a temperature of 0.3. It bridges the gap between hiring decisions 
   and candidate transparency, generating natural, encouraging, and detailed 
   recruiter scorecards explaining the reasoning behind the evaluation, highlighting 
   strengths, detailing weaknesses, and suggesting high-level areas for growth.

5. Recommendation Agent (recommendation_agent.py):
   Runs at a temperature of 0.35. It converts the evaluation results into a 
   hyper-personalized career growth plan. It identifies specific skill gaps, 
   recommends relevant courses and certifications, proposes hands-on project 
   concepts (complete with architecture, tech stack, and goals), outlines 
   interview preparation guides, and constructs a concrete, day-by-day 
   30-day learning roadmap.

6. FILESYSTEM MCP INTEGRATION
--------------------------------------------------------------------------------
The project leverages the Model Context Protocol (MCP) to manage all file read 
and write actions. Under this model, neither the Streamlit dashboard nor any 
of the individual ADK agents read or write to the filesystem directly. All file 
operations are gated by a secure FastMCP server (`filesystem_server.py`).

The client-server boundary operates over standard input/output (stdio) channels, 
making the service stateless, decoupled, and highly sandboxed. The MCP server 
exposes five distinct tools:
- `save_resume`: Takes base64-encoded PDF bytes from the UI and writes them securely.
- `read_resume`: Loads the file from `uploads/`, triggers the secure PDF text 
  extractor, and returns normalized, clean text.
- `save_report`: Accepts the aggregated JSON results from the Orchestrator and 
  writes a structured candidate report to `reports/`.
- `read_report`: Reads a previously generated JSON candidate report.
- `list_files`: Lists the files currently available in either `uploads/` or `reports/`.

7. SECURITY FEATURES
--------------------------------------------------------------------------------
To ensure suitability for enterprise environments, the system implements a strict, 
defense-in-depth security model across its MCP server and PDF parser:

- Directory Allowlisting: The MCP server maintains a strict allowlist. It only 
  accepts operations targeting the `uploads` and `reports` folders. Access to 
  arbitrary system directories is rejected.
- Path Traversal Prevention: Filenames provided by users are processed through 
  `os.path.basename(filename)` to strip any relative directories (e.g., `../`), 
  thwarting directory traversal attempts.
- Path Containment Verification: The server resolves absolute target paths and 
  verifies that they explicitly start with the permitted directory absolute path 
  using `.startswith(base_dir + os.sep)`.
- File Verification: The parser enforces a 5 MB file size limit before reading, 
  checks for the standard `%PDF-` file signature to prevent arbitrary file uploads, 
  and detects and rejects password-encrypted PDFs.
- Image-Only/Scanned PDF Detection: If text extraction yields fewer than 10 
  characters, the parser raises an exception, preventing the downstream pipeline 
  from running on empty or scanned image documents and prompting the user to 
  upload a text-searchable PDF.

8. STREAMLIT DASHBOARD
--------------------------------------------------------------------------------
The user interface is a premium, dark-mode Streamlit dashboard designed around 
modern web practices. Key design highlights include:
- Harmonious Dark Color Scheme: Set via `.streamlit/config.toml`, utilizing deep

  As the user initiates the pipeline, loading spinners guide them through each

- Symmetrical Grid Layouts: Interactive metrics, candidate summary scorecards,

  Streamlit's grid layouts.

  screen allows recruiters or developers to inspect the raw, strongly-typed JSON 
  returned by each of the four specialist ADK agents.

--------------------------------------------------------------------------------
9. AI RECRUITER WORKFLOW
--------------------------------------------------------------------------------
The system maps out a logical, real-world recruitment process flow:
1. Ingestion: The user uploads a candidate's resume PDF via the Streamlit UI, 
   which is saved to the secure `uploads/` folder by the MCP server.
2. Parsing & Classification: The Orchestrator triggers the MCP server to read 
   and extract text. The Resume Analysis Agent structures the text into standard 
   categories and determines the candidate's professional domain.
3. Scoring & Screening: The Recruiter Scoring Agent evaluates the profile using 
   industry-specific rules, outputting category grades and a hire/maybe/reject 
   decision.
4. Evaluation Capture: The Feedback Agent drafts a formal recruiter evaluation 
   detailing highlights and areas of concern.
5. Growth Advising: The Recommendation Agent creates a learning pathway and 
   a day-by-day 30-day action plan.
6. Persistence: The Orchestrator aggregates the results into a final report and 
   saves it to the `reports/` folder via the MCP server.
7. Presentation: The Streamlit UI displays the scorecard and career plan.

10. BUSINESS VALUE
--------------------------------------------------------------------------------
The AI Recruiter Agent Hub delivers measurable, multi-dimensional business impact:
- Enhanced Recruiter Efficiency: Shrinks initial candidate screening from 
  minutes to milliseconds. Recruiters can focus their attention on interviewing 
  and closing candidates rather than parsing text.
- Objective Evaluations: Downstream scoring is standard-based, ensuring every 
  candidate is graded consistently against their specific professional domain.
- Positive Candidate Branding: Instead of receiving a cold rejection, candidates 
  who do not progress are provided with constructive feedback and a concrete 
  30-day roadmap to help them upskill, improving the company's candidate net 
  promoter score (NPS).
- Architectural Scalability: Decoupled code modules, strict Pydantic schemas, 
  and stdio-based MCP protocols allow the system to easily scale to multi-resume 
  batch processing, external database backends, or automated pipeline integrations.

11. CHALLENGES SOLVED
--------------------------------------------------------------------------------
Several technical hurdles were addressed to deliver a robust, production-grade 
application:
- Async Event Loop Integration: Streamlit manages its own internal asyncio 
  event loop. Because Google ADK runs asynchronous processes natively, calling 
  `asyncio.run()` directly inside Streamlit handlers raises thread collision errors. 
  This was resolved by implementing a centralized loop runner utilizing 
  `nest_asyncio` to safely schedule and run ADK agents within Streamlit's environment.
- Model Overload Handlers (Gemini 503 Errors): During high concurrent loads, 
  the Gemini API can return 503 Service Unavailable errors. All agent modules 
  were wrapped in custom retry handlers with exponential backoff logic (retrying 
  up to 3 times with delays of 5s, 10s, and 20s), ensuring pipeline resilience.
- Image PDF Fallback: Handles non-standard scanned documents gracefully, raising 
  clean user-facing validation errors rather than allowing the model to hallucinate 
  on empty text.

   pipeline analytics over time.
