## 📝 Kaggle Writeup

### 1. Executive Summary & Problem Statement
Recruiters spend an average of **6 to 7 seconds** scanning a resume before making an initial screening decision. In high-volume recruitment channels, this creates a major bottleneck. Underqualified applicants clutter pipeline tracking systems, while exceptional candidates are frequently overlooked due to formatting discrepancies, a lack of optimized ATS keywords, or generic action verbs. Furthermore, rejected candidates are left in the dark, receiving automated generic rejection letters with zero constructive, actionable feedback, leading to a demoralizing applicant experience.

The **AI Recruiter Agent Hub** addresses this challenge by providing a secure, end-to-end, multi-agent evaluation pipeline. By leveraging the **Google Agent Development Kit (ADK)** and the **Model Context Protocol (MCP)**, the application automates initial screening for recruiters and offers deep, constructive career roadmaps for candidates. 

---

### 2. The Google ADK Multi-Agent Architecture
Rather than relying on a single, long-context LLM prompt that struggles with instruction-following and accuracy across varied tasks, the system splits the recruitment workflow into four specialized agents coordinate sequentially.

```
Streamlit UI
    │
    └──► Orchestrator Agent (orchestrator_agent.py)
              │
              ├── Stage 1: Resume Analysis Agent (Low Temp: 0.1)
              │     └── Output: structured JSON profile (ResumeSchema)
              │
              ├── Stage 2: Recruiter Scoring Agent (Low Temp: 0.2)
              │     └── Output: scorecards, hiring decision, flags (RecruiterScoreSchema)
              │
              ├── Stage 3: Feedback Agent (Medium Temp: 0.3)
              │     └── Output: strengths, weaknesses, tips (FeedbackSchema)
              │
              └── Stage 4: Recommendation Agent (Medium Temp: 0.4)
                    └── Output: 30-day week-by-week career plan (RecommendationSchema)
```

#### Specialist Roles & Configuration:
1.  **Resume Analysis Agent**: Configured with a low temperature of `0.1` for maximum determinism. It parses unstructured text from the PDF resume and extracts contact details, education history, work experience, skills, and projects into a strongly-typed schema.
2.  **Recruiter Scoring Agent**: Evaluates the candidate's structured profile with a temperature of `0.2` for nuanced judgment. It scores five categories (Technical, Projects, Experience, Communication, Education) on a 0–100 scale, estimates an ATS matching score, issues an overall score, flags gaps, and outputs a hiring decision (`"Hire"`, `"Maybe Hire"`, or `"Reject"`).
3.  **Feedback Agent**: Synthesizes the analysis and score card to generate strengths, weaknesses, and a prioritized resume-improvement checklist.
4.  **Recommendation Agent**: Delivers a personalized career growth plan, recommending targeted skills, courses, certifications, project ideas, and a structured week-by-week 30-day career roadmap.

#### Chaining & Orchestration:
The pipeline is governed by the **Orchestrator Agent** (`orchestrator_agent.py`). It enforces a strict data dependency chain: each agent receives the outputs of all previous stages as inputs. This design prevents hallucination, improves alignment, and maximizes reasoning coherence. 

The Orchestrator is implemented as a Python generator, enabling **progressive yielding**. Instead of making the user wait for the entire pipeline to execute, the UI dynamically renders each stage's loading status and dashboard outputs as soon as they are completed.

---

### 3. Secure Filesystem Integration via Model Context Protocol (MCP)
Security is a primary design driver in enterprise hiring systems. Direct file uploads pose significant risks, including malicious file modification and directory traversal attacks. 

To mitigate these threats, all file I/O operations are handled through a dedicated **Filesystem MCP Server** (`mcp/filesystem_server.py`) and **MCP Client** (`mcp/mcp_client.py`):
*   **Stdio Transport**: The MCP Client launches the server as a sandboxed subprocess and communicates exclusively via JSON-RPC over standard input/output.
*   **Directory Isolation**: The server strictly locks down path containment. It checks absolute paths and enforces `os.path.basename` extraction. Only reads from `uploads/` and writes to `reports/` are allowed. Any attempt to pass relative paths (e.g. `../../etc/passwd`) is immediately blocked.
*   **Automated PDF Parsing**: When a PDF is saved, the MCP Server reads it, parses the text content via PyPDF, and returns the normalized string back to the orchestrator, keeping raw file operations away from the agent execution layer.
*   **Report Persistence**: The final evaluation report (consolidated outputs from all four agents) is automatically written to `reports/` via the MCP server as a structured JSON record.

---

### 4. Interactive Streamlit UI & Responsive Presentation
The frontend dashboard (`app.py`) is styled using custom HSL colors and responsive CSS, built around two user journeys:
*   **Instant Demo Mode**: An offline simulator that demonstrates the pipeline for two candidates (**Sarah Chen**, Senior AI Engineer; and **Alex Rivers**, Backend Developer). When the recruiter clicks **"🧠 Run Demo Pipeline"**, the app loads pre-generated JSON files locally and runs them through a simulated progress generator. This allows judges and users to experience the complete workflow, including progressive `st.status` loading steps and full dashboards, without needing an API key.
*   **Upload Resume Mode**: The live production pipeline. Users upload their PDF, provide their Gemini API key (safely handled in Streamlit session state or pre-set in `.env`), and run the full multi-agent workflow.

The layout adapts gracefully to smaller screen viewports (mobile and tablet) by utilizing custom media queries and `auto-fit` flex-grids, ensuring metrics and timeline roadmap items stack neatly.

---

### 5. Real-World Business Value
*   **Recruiter Efficiency**: Automates the first-pass resume screen, saving hundreds of hours of manual labor by standardizing score cards and flagging red flags.
*   **Candidate Experience**: Every applicant—regardless of the hiring outcome—receives constructive, actionable feedback and a 30-day learning roadmap, elevating the employer's brand value.
*   **ATS Optimization**: Empowers candidates to identify clear skill gaps and resume issues, closing the mismatch loop in the job market.
*   **Enterprise-Ready Security**: The sandboxed MCP integration ensures that files are handled securely, satisfying standard security compliance requirements.
